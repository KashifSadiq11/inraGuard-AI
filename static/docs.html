🛡️ InfraGuard AI

A multi-agent DevOps intelligence platform powered by open-source AI. Analyzes logs, configs, deployments, and pipelines — instantly.

FastAPI Backend Groq Free AI 5 AI Agents 8 Analysis Tools Streaming UI 100% Free

What is InfraGuard AI?

InfraGuard AI is a backend application that uses AI agents to help DevOps engineers analyze infrastructure problems. Instead of spending hours reading logs or manually auditing configs, you paste your data and the AI gives you a structured report in seconds.

What problems does it solve?

  • Your production service crashed at 3am — what went wrong?
  • You wrote a Kubernetes config — is it secure?
  • You have a big git diff — is it safe to deploy?
  • Your CI/CD pipeline failed — what exactly broke?

🏗️ Architecture

The application is built with Python + FastAPI. It connects to Groq (free cloud AI) or Ollama (local AI) and exposes a REST API with a streaming web UI.

infraguard/
├── main.py              ← FastAPI app, all API endpoints
├── start.sh             ← One-click startup script
├── requirements.txt     ← Python dependencies
├── .env                 ← Your API keys (never committed to git)

│   ├── base.py          ← Agentic loop engine (tool calling + streaming)

    └── index.html       ← Dark theme web UI

🤖 5 AI Agents

Each agent is a specialist with a specific role, system prompt, and set of tools. They all use the same AI model but behave differently based on their instructions.

⚡
Commander (Auto)
Master agent. Routes to all specialists automatically. Best when you're not sure which agent to use. Uses all 8 tools.
🚨
Incident Triage
Analyzes crash logs and alerts. Produces root cause analysis, severity (P1-P4), and step-by-step runbook.
🔍
Config Auditor
Scans Kubernetes, Terraform, and Docker configs for security vulnerabilities and misconfigurations.
🚀
Deploy Risk
Analyzes git diffs before deployment. Gives a risk score 1-10, Go/No-Go decision, and rollback plan.
🏥
Pipeline Doctor
Diagnoses failing CI/CD pipelines. Finds the exact broken step and gives a fix with shell commands.

🔧 8 Analysis Tools

Example: Incident Triage

You paste:  "ERROR: Connection pool exhausted. FATAL: OOM kill signal"

Agent calls: parse_log_patterns()
             → finds 2 errors, OOM pattern, 4 total lines

Agent calls: extract_service_dependencies()
             → finds postgres:5432 mentioned

  Root Cause: Memory leak caused connection pool exhaustion
  Severity:   P1

  Runbook:    1. Check memory usage  2. Restart pod  3. Increase limits

Why Tool Calling Matters

Without tools, the AI just reads text and guesses. With tools, the AI gets structured data — exact error counts, line numbers, security findings — making the analysis much more accurate and specific.

📦 Installation

Requirements

  • Python 3.10 or higher
  • pip (Python package manager)
  • Internet connection (for Groq API)
  • Groq free account (console.groq.com)

Clone and Install

# Clone the repository
git clone https://github.com/KashifSadiq11/inraGuard-AI.git
cd inraGuard-AI

# Install dependencies
pip install -r requirements.txt

⚙️ Configuration

Create your .env file

cp .env.example .env
nano .env

.env file contents

# ── GROQ (Recommended — Free & Fast) ──
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile

# ── OLLAMA (Local fallback) ──
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:0.5b
ℹ️ If GROQ_API_KEY is set, the app uses Groq automatically. If not, it falls back to Ollama.

Get Groq API Key (Free)

  1. Go to console.groq.com
  2. Sign up with Google or GitHub (no credit card needed)
  3. Click API Keys → Create API Key
  4. Copy the key and paste it in your .env file

🚀 Running the Application

Start the server

./start.sh

Or manually

python3 -m uvicorn main:app --host 0.0.0.0 --port 8000

Open in browser

http://localhost:8000
✅ You should see the InfraGuard AI dashboard with the dark theme UI.

🖥️ Web UI Guide

Left Sidebar

  • Select Agent — Choose which specialist to use (or leave on Auto)
  • Optional Context — Add extra info like "production k8s manifest"
  • Input Box — Paste your logs, config, diff, or pipeline output
  • ⚡ Analyze Button — Start the analysis
  • Demo Scenarios — Pre-built examples to test the app

Right Panel

  • Status Bar — Shows which agent is running and which tools are being called
  • Output Area — Streams the AI analysis in real-time with markdown formatting
  • Tool Chips — Animated chips showing which tools the AI called
  • Copy Button — Copy the full analysis to clipboard

Top Header

  • Model Badge — Shows which AI model is active (e.g. llama-3.3-70b-versatile)
  • Status Badge — Online / Offline indicator

🐳 Ollama — Local AI Fallback

If you don't set a Groq API key, the app automatically uses Ollama running locally in Docker.

Your Ollama setup

Container: ollama/ollama
Port:      11434
Models:    qwen2.5:1.5b (better)
           qwen2.5:0.5b (faster)
⚠️ Ollama on CPU only is slow (~5 tokens/sec). Groq is recommended for daily use (~800 tokens/sec).

Pull a better model for Ollama

docker exec ollama ollama pull llama3.2
# Then update .env: OLLAMA_MODEL=llama3.2

❓ FAQ

Q: Is my data sent to the cloud?

If using Groq — yes, your input is sent to Groq's servers for AI processing. If using Ollama — everything stays 100% local on your machine.

Q: Can I use this with my real production logs?

Yes. Paste any log output directly. Be careful not to include passwords or secrets in the logs you paste if using Groq (cloud).

Q: How do I add more agents?

Open agents/specialists.py and add a new class that extends BaseAgent. Define a system_prompt and allowed_tools list. Then register it in the get_agent() function.

Q: How do I add more tools?

Open agents/tools.py, add a new Python function, register it in TOOL_REGISTRY, and add its definition to ALL_TOOL_DEFS in agents/base.py.

Q: The analysis is wrong or incomplete?

The 0.5b and 1.5b Qwen models are very small. Switch to Groq with llama-3.3-70b-versatile for much better quality analysis.

Q: How do I stop the server?

fuser -k 8000/tcp
